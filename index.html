Work Process

As an aspiring software engineer, I have begun to follow a very specific process in which I develop code. In the interest of productivity, following this method has surpassed my prior expectations of quality and efficiency.

1. Idea
2. Sketch
3. Design
4. Develop
5. Test
6. Launch

I begin by writing my ideas on a whiteboard and filtering out ones that are not feasible. I then proceed to sketching out a flow chart diagram similar to a UML diagram, highlighting all the inputs and outputs I have to work with. With all that in mind, I then start illustrating methods in which I can utilize these inputs and outputs in order to get the desired result. Once that is complete, I begin my favorite method: development. I develop the code systematically following my created design. Next, I continuously search for bugs until I am 100 percent confident that my code runs. Finally, once the code is bug-free, it is ready to be launched!
